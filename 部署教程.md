# Offkab Gallery 部署与使用教程

本项目是一个基于 Node.js 的轻量级本地画廊与资源爬虫系统。它可以自动从目标网站抓取指定的 FC2 项目信息及预览图，并在本地生成一个美观、现代化的网页画廊供您浏览和管理。

## 🛠️ 运行环境要求

在运行本项目之前，请确保您的电脑上已安装以下环境：

1. **Node.js**: 建议安装 v16 或以上版本。
   - 检查是否安装：在终端运行 `node -v`
   - 下载地址：[Node.js 官网](https://nodejs.org/)
2. **网络环境**: 
   - 爬虫需要能够正常访问 `sukebei.nyaa.si` 以及相关的图片托管网站。如果在某些地区无法直接访问，请确保您的终端/系统网络配置了代理。

---

## 📦 快速部署步骤

### 1. 下载项目代码
将整个项目文件夹（包含 `server.js`, `scraper.js`, `public/` 等）保存到您的本地电脑上。例如保存到 `/Users/用户名/Downloads/offkab`。

### 2. 安装依赖包
本项目依赖 `express` (用于网页服务) 和 `cheerio` (用于解析网页数据)。
请打开**终端 (Terminal/命令行)**，进入项目所在目录并安装依赖：

```bash
# 1. 进入项目目录（请将下面的路径替换为您实际的文件夹路径）
cd /Users/weiwei/Downloads/offkab

# 2. 初始化项目并安装必要的依赖
npm install express cheerio
```

### 3. 启动本地服务
依赖安装完成后，在终端中运行以下命令启动服务：

```bash
node server.js
```

如果看到以下提示，说明服务已经成功启动：
```text
🌐 Offkab Gallery Server running at:
   http://localhost:3000
```

### 4. 访问画廊
打开您的浏览器，在地址栏输入：
**[http://localhost:3000](http://localhost:3000)**

现在您就可以看到属于您自己的本地画廊了！

## 💡 功能使用指南

### 🔄 获取数据 (爬虫)
网页顶部导航栏提供了两种获取新项目的方式：
- **手动获取**：点击后，后台会启动一次爬虫任务，自动获取最新的 **20 个**新项目及照片，适合日常签到式更新。
- **自动获取**：开启后，后台会每隔 **1 小时**在后台默默运行一次无限制的爬取任务，保持您的画廊数据始终是最新的。

*(获取过程中，您可以点击“运行日志”按钮，在一个黑色的弹窗中实时查看后台正在下载什么图片)*

### 🔍 浏览与操作
- **搜索与筛选**：通过顶部的搜索框和分类标签（全部、有预览图、体积大、热门），快速找到您感兴趣的项目。
- **长图阅读**：在网页里直接**点击任意照片**即可全屏浏览。全屏模式下：
  - 按下键盘的 **`+`** 键或 **点击鼠标**：无极放大照片真实尺寸，并可以通过鼠标滚轮自由上下阅读高清长图细节。
  - 按下键盘的 **`-`** 键：缩小照片。
- **复制磁力链**：点击“🧲 复制磁力链”按钮，即可一键复制磁力链接，并在屏幕下方弹出成功提示。
- **黑名单机制**：遇到不喜欢、不想保留的项目，可以点击“🚫 拉黑”按钮。
  - 确认后，项目会立刻在网页上消失。
  - **同时会自动删除本地硬盘上的照片数据**以节省空间。
  - 该项目会被永久拉黑，以后的爬虫任务都会智能跳过它。

## 📂 目录结构说明

如果您想了解项目的内部结构，这里是一份简单的清单：

```text
offkab/
├── server.js          # Web 核心服务端 (提供网页访问和 API 接口)
├── scraper.js         # 爬虫核心程序 (负责抓取网页和下载图片)
├── public/            # 网页前端文件存放区
│   ├── index.html     # 画廊主页结构
│   ├── style.css      # 画廊样式 (暗黑模式/动画等)
│   ├── data.json      # 核心数据库文件 (所有项目的文本信息均保存在此)
│   ├── blacklist.json # 黑名单库 (被拉黑的 ID)
│   └── images/        # 下载的本地高清照片存放目录
└── 部署教程.md          # 也就是您现在正在阅读的这份文档
```

**遇到问题？**
如果终端报错或者网页无法打开，请检查：
1. `node server.js` 终端进程是否被意外关闭？(必须保持终端运行)
2. 端口 `3000` 是否被其他程序占用？
3. 网络代理是否设置正确，导致爬虫无法获取到图片？
